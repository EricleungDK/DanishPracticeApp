# 2026-03-18

## Summary

Resolved critical packaging and translation issues. Fixed fill-in-the-blank exercise translations by adding English answer fields, configured Windows executable packaging via Electron Forge, and solved a major sql.js bundling blocker through Rollup hook rewrites.

## Key Changes

- **Fill-blank translations**: Added `answer_en` field to `FillBlankAnswerKey` type; updated 30 templates in `fill-blank.json` with English translations; modified `explanation-generator.ts` to use English words in translation explanations instead of Danish
- **Windows packaging**: Configured `forge.config.ts` with MakerSquirrel; added `postMake` hook to copy `Setup.exe` to project root; added `make:win` npm script for convenience
- **sql.js bundling fix**: Resolved Emscripten CJS incompatibility by adding Rollup `renderChunk` hook to rewrite `require("sql.js")` → `require("./sql-wasm.js")`; added `closeBundle` hook to copy `sql-wasm.js` and `sql-wasm.wasm` into `.vite/build/`
- **Startup error visibility**: Wrapped `app.on('ready')` in try/catch with `dialog.showErrorBox()` so errors surface in UI instead of failing silently
- **Housekeeping**: Updated `.gitignore` to exclude `*Setup*.exe` files

## Technical Decisions

The sql.js issue required keeping it external to Vite (since bundling fails) while still making it accessible inside the asar archive. The renderChunk rewrite approach allows the compiled code to dynamically load the correct module path at runtime without requiring post-asar rewrites.
