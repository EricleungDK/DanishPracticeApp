### Score History Chart Fix
- Chart was showing newest date on the LEFT because the DB query used `ORDER BY started_at DESC LIMIT 20` and the result flowed unchanged into Recharts
- Subagent suggested flipping `DESC`ŌåÆ`ASC` but that combined with `LIMIT 20` would return the oldest 20 sessions ever, not the latest 20
- Correct fix: keep `DESC LIMIT 20` then `.reverse()` the array before returning ŌåÆ latest 20 rows in chronological order

### Pause & Resume for Practice / Review
- User feedback: no way to pause mid-session and come back later without losing progress
- Added `Pause & go back` button at top-left of `Exercise.tsx` and `Review.tsx`
- New store fields: `sessionType: 'exercise' | 'review' | null`, `pausedSession: SessionSnapshot | null`
- New actions: `pauseSession`, `resumePausedSession`, `loadPausedSession`
- Snapshot (exercises array, currentIndex, sessionStats, lastResult, awaitingRating) persisted to the existing `settings` KV store under key `paused_session` as JSON
- `App.tsx` calls `loadPausedSession()` on mount, so paused state survives app restart
- `startPractice`/`startReview` check `pausedSession.sessionType` first ŌĆö auto-resume if match, else fetch fresh
- Sidebar now uses `sessionType` (not `currentPage`) to route correctly, fixing an existing bug where going to Dashboard mid-Review then clicking Review would start a NEW review instead of returning to the in-memory one
- Sidebar shows green `live` badge for active sessions and orange `paused` badge when a session is paused

## Technical Details

### Implementation Highlights
- **Single-slot paused session**: Only one paused session can exist at a time. If user pauses a Practice then later pauses a Review, the Practice snapshot is overwritten. Simpler than dual-slot, acceptable for V1
- **Session type tracking**: Added `sessionType` to the store so Sidebar can correctly distinguish an in-memory Practice session vs an in-memory Review session (they both use the same `sessionExercises` array)
- **Persistence via existing settings IPC**: No new IPC/preload/DB changes needed ŌĆö `saveSetting('paused_session', JSON.stringify(snapshot))` reuses the existing key-value settings table
- **Resume on startPractice/startReview**: Check for matching paused type before fetching fresh exercises. Clean, no extra UI
- **Mid-rating pause preserved**: Snapshot includes `lastResult` and `awaitingRating`, so resuming mid-rating returns user to the rating UI for the same exercise

### Challenges Overcome
- Subagent's proposed chart fix (flip DESCŌåÆASC) would have broken the LIMIT semantics. Caught by reading the query carefully before applying
  - Solution: kept DESC+LIMIT, added `.reverse()` on the result
- Sidebar had pre-existing bug: `currentPage === 'review'` guard meant leaving mid-review then clicking Review started a new session
  - Solution: use `sessionType` field instead of `currentPage` since `currentPage` changes when user navigates away

### Design Decisions
- **Option A (remove) for Skill Breakdown** instead of Option B (bar chart) or Option C (topic drilldown)
  - Rationale: user said "I don't see the real use of it yet" ŌĆö simplest remedy is to remove it rather than reinvest in a feature they don't value
- **Full removal of the backing IPC chain** rather than leaving dead code
  - Rationale: CLAUDE.md says "if you are certain that something is unused, you can delete it completely" ŌĆö avoid dead code accumulation
